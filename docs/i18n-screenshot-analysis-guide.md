# 国际化截图分析指南

## 📸 截图提供指南

### 推荐工具
- **macOS**: CleanShot X, Snagit, Xnapper
- **Windows**: ShareX, Snagit
- **跨平台**: Flameshot

### 截图要求
1. **完整页面截图** - 包含从顶部到底部的所有内容
2. **高分辨率** - 至少 1920x1080，推荐 2560x1440
3. **清晰可读** - 文字清晰，无模糊
4. **标注问题区域**（可选）- 使用红框或箭头标注可疑区域

### 截图命名规范
```
{locale}-{page}-{timestamp}.png

示例：
en-home-20250130.png
zh-about-20250130.png
en-contact-issue-20250130.png
```

## 🔍 AI 分析流程

### 第一步：提供截图
将截图拖放到对话中，并说明：
- 页面语言（en/zh）
- 页面路径（如 `/en/contact`）
- 关注点（如"检查是否有中文内容"）

### 第二步：AI 深度分析
AI 将检查：
1. **语言一致性** - 页面内容是否与语言设置一致
2. **混合内容** - 是否存在语言混合
3. **翻译质量** - 翻译是否准确、自然
4. **UI 元素** - 按钮、标签、提示文本等
5. **错误信息** - 错误提示是否国际化
6. **日期时间** - 格式是否符合语言习惯

### 第三步：问题定位
AI 将提供：
- 问题截图标注
- 具体文件路径
- 代码行号
- 修复建议

## 🎯 常见问题区域

### 高风险区域
1. **错误边界** - `layout.tsx` 错误提示
2. **表单验证** - 客户端验证消息
3. **加载状态** - Loading 文本
4. **空状态** - Empty state 文本
5. **Toast 通知** - 成功/失败提示

### 中风险区域
1. **页脚** - 版权信息、链接
2. **导航** - 菜单项、面包屑
3. **按钮** - CTA 按钮文本
4. **标签** - 表单字段标签

### 低风险区域
1. **技术术语** - API、JSON、HTTP 等
2. **品牌名称** - 公司名、产品名
3. **代码示例** - 代码块内容

## 🔧 快速修复指南

### 错误边界国际化
```typescript
// ❌ 错误
<div>监控组件加载失败</div>

// ✅ 正确
<div>{t('errors.monitoringFailed')}</div>
```

### 表单验证国际化
```typescript
// ❌ 错误
const error = "请输入有效的邮箱";

// ✅ 正确
const error = t('validation.invalidEmail');
```

### Toast 通知国际化
```typescript
// ❌ 错误
toast.success("提交成功");

// ✅ 正确
toast.success(t('messages.submitSuccess'));
```

## 📈 质量标准

### 优秀（90-100分）
- 无语言混合
- 所有文本国际化
- 日期时间格式正确
- 错误提示完整

### 良好（80-89分）
- 极少语言混合（≤2处）
- 主要内容国际化
- 日期时间基本正确

### 需改进（<80分）
- 明显语言混合（>2处）
- 部分内容未国际化
- 日期时间格式不一致

## 🚀 使用示例

### 示例 2：对比中英文页面
```
用户：对比这两个截图，检查翻译一致性

AI：对比分析...
发现 2 个问题：
1. 英文页面"Business Hours"在中文页面翻译为"营业时间"，建议统一为"工作时间"
2. 中文页面缺少"Emergency Contact"对应翻译
```

## 📝 注意事项

1. **截图清晰度** - 确保文字可读
2. **完整性** - 包含所有关键区域
3. **上下文** - 提供页面路径和语言信息
4. **标注** - 可选，但有助于快速定位
5. **隐私** - 移除敏感信息（如真实邮箱、电话）
